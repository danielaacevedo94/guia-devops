Construcción

Descripción de la Actividad

En esta actividad se realiza la compilación, enlace o términos similares referentes a crear el empaquetado de la solución. Esta actividad depende de las instrucciones propias del lenguaje de programación y es una tarea repetitiva que se puede automatizar. (Perera et al., 2017).

Se busca integrar y compilar los códigos realizados para garantizar que los cambios no han impedido esta labor. (Sousa et al., 2019)

La construcción integra las nuevas funcionalidades que se codificaron y crea los nuevos artefactos añadiendo las nuevas funcionalidades. Así como las pruebas permiten asegurar que tanto las nuevas funcionalidades como las antiguas se ejecutan correctamente. (Badshah et al., 2020).

Descripción de las categorías de las herramientas de la Actividad

Para la Construcción se utilizan herramientas de compilación o empaquetado que permiten automatizar el proceso de convertir el código fuente en un programa ejecutable o en otro formato intermedio, es decir, para convertir el código fuente escrito en un lenguaje de programación, como C++, Java, o Python, en un formato ejecutable, como un archivo .exe en Windows armando la unidad desplegable. A continuación se describe:

  • Compilación o empaquetado

    La compilación es el proceso de traducción de un código fuente (escrito en un lenguaje de programación de alto nivel) a lenguaje máquina (código objeto) para que pueda ser ejecutado por la computadora (Definicion.de, 2024a; phenobarbital, 2010).
    En términos generales, el empaquetado es el proceso de almacenar o preparar un producto para su transporte y/o venta. Consiste en la acción de guardar o envolver el producto de tal manera que esté listo para su traslado, exhibición o consumo. (Definicion.de, 2024b).
    En el desarrollo de software, es el proceso que permite que la aplicación funcione en el dispositivo deseado, computadora o un dispositivo móvil o incluso un entorno de producción o un servidor de prueba (Thales, 2023). En (NinjaOne, 2024) lo refieren como varios productos de software empaquetados juntos, el usuario final suele verlo como una entidad única, que pueden estar compuestos por varias aplicaciones informáticas distintas.
    Algunas funcionalidades que se pueden encontrar en las herramientas de compilación o empaquetado son las siguientes:

    • Compilación de código fuente.
    • Gestión de Dependencias: Manejan y resuelven las dependencias del proyecto, descargando y organizando las bibliotecas necesarias para que el software funcione correctamente.
    • Empaquetado de la aplicación: Crean paquetes de software listos para ser distribuidos, lo que incluye la preparación de instaladores y la generación de contenedores o archivos ejecutables​.
    • Automatización de la compilación: Permiten compilar automáticamente el código fuente en ejecutables o bibliotecas, asegurando que el software se construya de manera consistente cada vez.
    • Integración con repositorios de código:
    • Integración con herramientas de pruebas: Se integran para pruebas unitarias, de integración y de aceptación para validar que el código funciona como se espera antes de ser desplegado​.
    • Integración con herramientas de despliegue: Para facilitar la implementación automática del software en diferentes entornos (desarrollo, pruebas, producción) mediante scripts o configuraciones predefinidas​.
    (Mani, 2024; Swimm, 2023; Vladasel, 2022)
