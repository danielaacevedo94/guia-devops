{% trans Descripción de las categorías de las herramientas de la Actividad %}

{% blocktranslate %}Para la Codificación se utilizan software de control de versiones, editores de código e IDEs de programación pues permite a los equipos escribir código, colaborar en el mismo, minimizando así, los conflictos y facilita la práctica de integración continua. A continuación, se describen estos:{% endblocktranslate %}

  • {% trans Control de versiones %}

    {% blocktranslate %} Los sistemas de control de versiones son herramientas que actúan como repositorios centralizados, facilitando que todos los miembros del equipo de desarrollo tengan acceso a los cambios (propiedad colectiva del código). Además, con ellas se logra lo siguiente:

    • Almacenan todos los cambios en un historial para permitir su trazabilidad.
    • Se puede revertir rápidamente a una versión estable anterior en caso de error durante la integración.
    • Permite una recuperación de fallas sin demoras significativas.
    {% endblocktranslate %} (Pastrana-Pardo et al., 2022)

  • {% trans Entornos de desarrollo integrados (IDE) %}

    {% blocktranslate %} Es una aplicación de software que ayuda a los programadores a desarrollar código de software de manera eficiente. Incluyen funcionalidades que van más allá de la edición de texto y proporcionan una interfaz central para herramientas de desarrollo comunes (AWS, 2023). Entre ellas incluyen:

    • Automatización de la edición de código: Permite realizar tareas repetitivas de edición de código automáticamente, como formateo o alineación de código, lo que ahorra tiempo y reduce errores humanos.
    • Resaltado de sintaxis: Mejora la legibilidad del código al aplicar diferentes colores a palabras clave, variables y estructuras del lenguaje de programación, facilitando la identificación de errores y la comprensión del código.
    • Finalización de código inteligente: Proporciona sugerencias de autocompletado basadas en el contexto y el análisis del código, acelerando el proceso de escritura y reduciendo los errores tipográficos.
    • Refactorización del soporte: Permite reorganizar y modificar el código sin cambiar su comportamiento externo, ayudando a mejorar su estructura, legibilidad y mantenibilidad.
    • Automatización de la creación local: acilita la ejecución de tareas de desarrollo como compilación o pruebas en el entorno local del desarrollador, agilizando el flujo de trabajo​.
    • Compilación: Traduce el código fuente en un ejecutable o en un formato que la máquina pueda entender, asegurando que el software funcione correctamente en diferentes entornos​.
    • Pruebas: Ejecuta automáticamente pruebas unitarias, de integración, y funcionales para verificar que el software funciona como se espera antes de su despliegue.
    • Depuración: Proporciona herramientas para identificar, rastrear y corregir errores en el código, mejorando la estabilidad y funcionalidad del software.
    {% endblocktranslate %} (Mani, 2024; Swimm, 2023; Vladasel, 2022)

  • {% Gestión de artefactos %}

    {% blocktranslate %} Los artefactos son aquellos elementos tangibles que son producidos durante la creación de un proyecto {% endblocktranslate %}(Universidad Tecmilenio, 2020).
    {% blocktranslate %} Los artefactos de DevOps son referenciados en una etapa del pipeline para el despliegue automatizado en el entorno de destino y se utilizan para especificar las versiones de los paquetes de software a desplegar. Estos artefactos pueden incluir una imagen de contenedor, un artefacto genérico, un gráfico de Helm, o definiciones en línea, y su origen varía según el tipo de artefacto. En DevOps, se pueden crear referencias a los siguientes tipos de artefactos {% endblocktranslate %}(Oracle, 2024):

      {% blocktranslate %}
    • Repositorio de imágenes de contenedor: Se refiere a las imágenes de Docker.
    • Configuración de despliegue del grupo de instancias: Se refiere a un archivo en formato YAML que se utiliza para gestionar el despliegue del grupo de instancias, definiendo los artefactos a descargar y su ubicación en la instancia.
    • Manual de Kubernetes.
    • Artefacto general: Se refiere a cualquier paquete de software genérico que se vaya a desplegar.
    • Gráfico de Helm: Se refiere a los gráficos de Helm.
    • Especificación de comando de Helm: Se refiere a un conjunto de comandos de Helm que se ejecutan directamente durante el despliegue.
    • Especificación de comando: Se refiere a un conjunto de comandos y pasos personalizados que se ejecutan en un pipeline de despliegue.
      • {% endblocktranslate %}
    {% blocktranslate %} Las herramientas para ello según (JFrog, 2024) funcionan como un centro centralizado para DevOps, integrándose con herramientas y procesos para mejorar la automatización, aumentar la integridad e incorporar las mejores prácticas en cada etapa. Desde una perspectiva arquitectónica, es un gran facilitador de eficiencia. Contar con la capacidad de almacenar esto en un repositorio centralizado y bien gestionado, al que todos puedan acceder bajo ciertos controles, y que incluya medidas de seguridad y etiquetado, es esencial para que sea lo más ágil y rápido posible.
    A continuación, se describen algunas de estas características generales:
    • Almacenamiento Centralizado y Gestión de Versiones: Estas herramientas ofrecen un repositorio centralizado para almacenar y gestionar diferentes versiones de artefactos, como binarios, librerías, y contenedores. Esto asegura que todos los desarrolladores accedan a los mismos artefactos, reduciendo la duplicación y garantizando la consistencia en los entornos de desarrollo, prueba y producción.
    • Soporte para Múltiples Formatos de Paquetes: Las herramientas de gestión de artefactos soportan una amplia gama de formatos de paquetes (como Maven, npm, Docker, y NuGet), lo que las hace versátiles para diferentes tipos de proyectos. Esto permite a los equipos manejar artefactos de manera uniforme, independientemente del lenguaje de programación o plataforma utilizados.
    • Integración con Pipelines CI/CD: Estas herramientas están diseñadas para integrarse de forma fluida con pipelines de Integración Continua/Despliegue Continuo (CI/CD). Esto permite la automatización del flujo de trabajo desde la construcción hasta la implementación, asegurando que los artefactos correctos se utilicen en cada etapa del proceso.
    • Seguridad y Control de Acceso: Proporcionan controles de seguridad avanzados para proteger los artefactos, incluyendo el escaneo de vulnerabilidades y la gestión de permisos. Esto es esencial para prevenir la introducción de componentes inseguros en el pipeline de software y para cumplir con las normativas de seguridad.
    • Escalabilidad y Distribución Global: Estas herramientas están diseñadas para ser escalables, permitiendo la gestión de grandes volúmenes de artefactos en proyectos de cualquier tamaño. Algunas también ofrecen capacidades de replicación y distribución global, asegurando que los artefactos estén disponibles en múltiples ubicaciones geográficas para equipos distribuidos.
      • {% endblocktranslate %}
    (JFrog, 2019; Mirren McDade & Laura Lannini, 2024; Plutora, 2023)
