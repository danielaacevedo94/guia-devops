Plan

Activity description

DevOps teams are responsible for conceptualizing and designing the software they will develop, identifying client needs and proposing solutions through the development or improvement of the software. In this process, they detail the software's functionalities and features. They also plan which tools to use so the team understands pending tasks and progress, and assesses whether they can meet established deadlines, among other aspects (Perera et al., 2017).

(Sousa et al., 2019) mentions that the purpose of planning is to understand what needs must be met to obtain value from the information provided throughout the cycle. This encourages continuous improvement.

With regard to planning, it seeks to establish the minimum set of functionalities that add value in each iteration. It also analyzes and evaluates the metrics or data provided by monitoring to establish improvements for the next iteration (or stage). It also seeks to establish the acceptance criteria to be met and develop a plan to achieve the stated objectives. It also allows for defining what activities will be performed and by whom, in addition to designing the features to be coded (Badshah et al., 2020).

Purpose: Understand what needs must be met to obtain value.

Objectives:

  • Establish the minimum set of functionalities that add value in each iteration.
  • Analyze and evaluate the metrics or data provided by monitoring to establish improvements in the next iteration (or stage).
  • Establish the acceptance criteria to be met.
  • Develop a plan to achieve the stated objectives.
  • Define which activities each person will perform.
  • Design the features to be coded.
